# Life RPG

Геймификация жизни: прокачка PM/Product/Дисциплины, форма, квесты спринта,
аватар, который меняется вместе с тобой. Vite + React + TypeScript + Tailwind.
Персонаж собирается из журнала дней в репозитории (`data/days.json`), поверх
которого лежит черновик текущего дня в браузере (localStorage). Плюс
экспорт/импорт JSON-сохранения.

## Локальный запуск

Нужен установленный Node.js (LTS, nodejs.org).

```bash
npm install
npm run dev
```

Откроется на `http://localhost:5173`.

## Деплой на Vercel через GitHub

1. **Создай GitHub-аккаунт**, если его нет — github.com → Sign up.
2. **Создай пустой репозиторий** (например `life-rpg`), без README — он уже есть.
3. В папке проекта:
   ```bash
   git init
   git add .
   git commit -m "Life RPG: первая версия"
   git branch -M main
   git remote add origin https://github.com/ТВОЙ_НИК/life-rpg.git
   git push -u origin main
   ```
4. Зайди на **vercel.com** → Sign up через GitHub → **Add New → Project** →
   выбери репозиторий `life-rpg` → **Deploy**. Vite определится автоматически,
   ничего настраивать не нужно. Через минуту получишь ссылку вида
   `life-rpg.vercel.app`.
5. На телефоне открой ссылку в Safari/Chrome → **«Добавить на домашний экран»**.
   Получится иконка и полноэкранный запуск без адресной строки (PWA-манифест
   уже настроен в `public/manifest.webmanifest`).

Дальше — просто редактируешь код и пушишь (`git push`), Vercel передеплоит
автоматически.

## Журнал дней (`data/days.json`)

Источник правды по персонажу — массив записей по дням, лежащий прямо в
репозитории. Каждая запись — уже посчитанный за день gross-XP плюс показатели
дня:

```json
{
  "date": "2026-07-15",
  "note": "Вторник",
  "xp": { "pm": 90, "prod": 45, "disc": 50 },
  "fit": 50,
  "quests": { "wake": 1, "course": 1 },
  "sleep": 7.5,
  "calories": 2180,
  "mood": 4
}
```

Все поля, кроме `date`, необязательны. При загрузке приложение вызывает
`replay(days)` из `src/engine/engine.ts`: берёт стартового персонажа
(`createInitialState`) и прогоняет записи по датам подряд по тем же правилам,
что и живое приложение — начисление XP, ежедневное содержание (upkeep), откат
уровня, дно «пик − 15», заморозка. `quests` двигают только прогресс спринта, XP
повторно не начисляют (он уже в `xp`/`fit`). Добавил день — закоммитил файл,
персонаж пересчитался.

Кнопки быстрой активности (квесты) и правка тела пишут поверх журнала
**черновик текущего дня** в localStorage — он живёт до тех пор, пока день не
попал в `data/days.json`.

## Что где лежит

- `src/engine/rules.ts` — вся игровая математика в одном месте: формулы XP,
  содержание навыков (upkeep), кривые формы, веса героя. Хочешь жёстче
  падение — крути коэффициенты здесь.
- `src/engine/engine.ts` — чистая логика (закрытие дня, начисление/списание
  XP, откат уровня, идле-дебафф). Без React, можно переиспользовать хоть в
  Telegram-боте.
- `src/engine/actions.ts` — применение действий (тап квеста, итог дня и т.д.)
  к состоянию. Общий код для браузера и серверных функций.
- `src/engine/store.tsx` — React state + localStorage + облачная синхронизация.
- `api/state.ts`, `api/log-day.ts` — серверные функции Vercel для облачной
  синхронизации (см. раздел ниже).
- `src/components/` — экраны «Персонаж» и «Дашборд».
- `src/assets/avatar/street.png` и `office.png` — твой аватар (уличный тир /
  офисный тир, разблокируется на 25 уровне Product). Сейчас это временные
  заглушки — просто замени эти два файла своими картинками с тем же именем,
  ничего в коде менять не нужно.

## Как устроена механика (кратко)

- Все навыки и форма — шкала **1–100**.
- Уровень не только растёт, но и падает: у каждого навыка есть **содержание**
  (upkeep) — XP, которое списывается каждый день просто за то, что ты стоишь
  на месте. Чем выше уровень, тем дороже содержание. Не заработал достаточно
  — уровень откатывается. Есть дно: не ниже (пик − 15).
- **Активность за 7 дней** (дуга на экране персонажа) — сумма всего
  заработанного XP за неделю. Ниже 25/100 — включается дебафф «Простой»:
  содержание навыков растёт на 40%.
- **Форма** = 50% регулярность (XP тренировок за 14 дней) + 50% тело и
  результаты (подтягивания/отжимания/% жира по кривой). Вес отображается
  только как тренд, XP не начисляет и не забирает.
- **Заморозка** — тумблер на экране персонажа: содержание не списывается, но
  и XP не капает. Используй на время болезни/отпуска.
- **Квесты спринта** — живут 21 день, тапаешь «+»/«−» для прогресса, XP
  начисляется сразу.
- **Общий уровень героя** = 30% PM + 25% Product + 20% Форма + 15%
  Дисциплина + 10% Активность.

## Экспорт/импорт сохранения

На экране «Дашборд» внизу — «Скачать сохранение» (JSON) и «Загрузить».
Работает всегда, независимо от облачной синхронизации ниже.

## Облачная синхронизация (чтобы Claude мог менять состояние из чата)

По умолчанию сохранение живёт только в localStorage телефона — ни с одного
другого устройства (и уж тем более из чата с Claude) его не поменять. Чтобы
можно было написать Claude «сегодня работал 6 часов, час курса, зал был» и
увидеть это в приложении при следующем открытии — нужна общая база данных.
Это подключается один раз:

### 1. Supabase (бесплатная база данных)

1. Зарегистрируйся на supabase.com (аккаунт создаёшь сам).
2. Create a new project (любое название и пароль от БД — их не потребуется
   вспоминать, они не используются нигде дальше).
3. В левом меню открой **SQL Editor** → **New query**, вставь содержимое
   файла `supabase-setup.sql` из этого репозитория → **Run**. Это создаст
   таблицу `game_state` с одной строкой (твоё сохранение) и включит RLS
   (Row Level Security) без публичных политик — то есть снаружи её никто не
   прочитает и не изменит, кроме как через service-role ключ, который
   остаётся только на сервере (см. ниже).
4. В **Project Settings → API** скопируй:
   - **Project URL** (`https://xxxx.supabase.co`)
   - **service_role** ключ (секретный, длинная строка) — не путать с anon
     ключом, нужен именно service_role.

### 2. Придумай секрет для API

Придумай любую длинную случайную строку — это будет пароль, которым будешь
делиться только с Claude (в чате), чтобы разрешить менять твоё сохранение.
Например: `life-rpg-secret-8f3k2m9x` (лучше сгенерировать что-то подлиннее).

### 3. Переменные окружения в Vercel

Открой свой проект на vercel.com → **Settings → Environment Variables** и
добавь четыре штуки (для окружения Production, можно и для Preview):

| Имя | Значение |
|---|---|
| `SUPABASE_URL` | Project URL из шага 1 |
| `SUPABASE_SERVICE_ROLE_KEY` | service_role ключ из шага 1 (секрет, только сервер) |
| `LOG_API_SECRET` | твой секрет из шага 2 |
| `VITE_APP_SECRET` | тот же самый секрет из шага 2 (нужен клиенту в браузере) |

После сохранения нажми **Deployments → ⋯ → Redeploy** на последнем деплое
(или просто сделай `git push` — env-переменные подхватятся при следующей
сборке).

### 4. Как это работает дальше

- Приложение на телефоне при открытии само подтягивает свежее состояние с
  сервера (см. надпись «☁️ синхронизировано» под уровнем героя).
- Каждый тап по квесту или сохранение «Итога дня» уходит и в Supabase тоже.
- Когда пишешь Claude в чате «сегодня было X, Y, Z» — Claude пересчитывает
  XP по тем же формулам из `src/engine/rules.ts` и вызывает
  `POST /api/log-day` на твоём задеплоенном домене с секретом из шага 2.
  Изменения появятся на телефоне при следующем открытии/обновлении страницы.
- Если облако недоступно (нет интернета) — приложение продолжает работать
  локально через localStorage, надпись сменится на «⚠️ офлайн».

Без этой настройки (шаги 1–3) приложение работает как раньше — полностью
локально, просто без синхронизации с чатом.
